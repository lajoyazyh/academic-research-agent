## 项目概述

### 项目定位

面向学术文献调研的 LLM Agent 应用。给定一个研究主题，Agent 自主规划检索策略，从 arXiv / Semantic Scholar / Crossref / OpenAlex 等学术数据库搜索前沿论文，阅读摘要和全文，记录结构化笔记，最终生成万字级文献综述。

### 核心价值

- **自动化调研**: 从关键词规划到综述生成的全流程自动化
- **交互式体验**: 支持用户在关键节点介入，调整策略
- **混合来源**: Agent 搜索 + 用户上传，灵活组合
- **RAG 增强**: 基于检索增强的对话和笔记生成

### 运行模式

1. **CLI 命令行模式**: 适合批量处理和脚本化任务
2. **Web 交互式平台**: 适合可视化操作和实时监控

## 技术架构

### 核心架构：三种推理框架的有机融合

```
┌─────────────────────────────────────────────────┐
│  阶段 0: Plan-and-Execute（规划 → 执行）          │
│  Agent 先产出一个显式的研究计划：                  │
│  关键词拆分 → 数据库选择 → 备选策略 → 预期深度     │
├─────────────────────────────────────────────────┤
│  阶段 1-N: ReAct（推理 → 行动循环）               │
│  [思考] → 选择工具 → [执行] → 观察结果 → [再思考]  │
├─────────────────────────────────────────────────┤
│  内置: Reflexion（自我反思，5 个层次）             │
│  L1: 单次错误即时反馈（JSON格式/未知工具/运行异常） │
│  L2: 连续 3 次同类错误 → 深度反思提示              │
│  L3: FINISH 前自主质检（Self-Critique）            │
│  L4: PDF 自动兜底下载                             │
│  L5: 论文列表自动传递到前端                        │
└─────────────────────────────────────────────────┘
```

### 技术栈

| 类别 | 技术 | 用途 |
|------|------|------|
| **后端框架** | FastAPI | Web API 服务 |
| **LLM** | Zhipu AI (glm-4-flash) | 推理和生成 |
| **Embedding** | Zhipu AI (embedding-2) | 向量检索 |
| **PDF 处理** | PyMuPDF (fitz) | PDF 文本提取 |
| **检索** | sklearn (BM25) | 对话栏检索 |
| **向量计算** | numpy | 余弦相似度 |
| **前端** | 原生 JavaScript + Vditor | Web UI |
| **测试** | pytest | 单元测试 |

### Session 状态机

```
planning → plan_confirmed → searching → search_complete
→ reviewing_notes → writing → reviewing_draft → complete
```

**状态说明**:
- `planning`: 生成关键词规划
- `plan_confirmed`: 用户确认关键词
- `searching`: Agent 搜索论文
- `search_complete`: 搜索完成，等待审核
- `reviewing_notes`: 用户审核笔记
- `writing`: 撰写综述
- `reviewing_draft`: 用户审核综述草稿
- `complete`: 完成

**特殊机制**:
- 任意状态可跳回 `searching`（追加调研）
- 自动修复：加载 Session 时检测卡住状态，若已有草稿则自动修复为 `complete`

## 已完成功能

### ✅ 核心功能

#### 1. 对话 RAG (BM25 检索)
- **功能**: 在对话栏提问时，自动从已下载的 PDF 中检索相关段落
- **实现**: `tools/retriever.py` (BM25Retriever)
- **特点**:
  - 实时检索，返回 Top-5 相关段落
  - 前端显示检索状态（📚 已检索 N 个段落）
  - 检索结果注入到 LLM prompt 中

#### 2. 论文注册一体化
- **功能**: 下载 PDF 并自动注册到 papers_list.json
- **实现**: `tools/paper_register.py` (PaperRegisterTool)
- **解决的问题**: 之前 Agent 使用 `arxiv_pdf_reader` 下载 PDF 但未注册，导致论文列表不完整
- **特点**:
  - 审核摘要 → 收录论文，一步完成
  - 下载失败不注册，保证数据一致性

#### 3. 搜索取消功能
- **功能**: 在搜索进行中可随时停止
- **实现**: `web_app.py` (POST /api/sessions/{id}/run/cancel)
- **特点**:
  - 前端显示"停止检索"按钮
  - 支持从任意状态返回 `searching`
  - 自动恢复卡住的 Session

#### 4. Trace 管理
- **功能**: 记录 Agent 执行轨迹，支持多次搜索
- **实现**: `backend/session_manager.py` (save_traces with append=True)
- **特点**:
  - 每条 trace 带时间戳
  - 多次搜索自动追加，带 SECTION 分隔
  - 前端 TOC 导航，快速跳转

#### 5. RAG 笔记生成
- **功能**: 基于全 PDF 的深度笔记生成
- **实现**: `tools/rag_note_generator.py` (RAGNoteGenerator)
- **特点**:
  - 6 段式结构：研究背景、核心方法、实验设置、关键结果、消融与分析、亮点与不足
  - Embedding 向量检索 Top-5 段落
  - Fallback 策略：BM25 → 摘要生成
  - 每段最少字数保证

#### 6. 质量门禁
- **功能**: 确保 Agent 收录足够论文
- **实现**: `core/agent.py` (质量门禁检查)
- **特点**:
  - 默认最少 3 篇（可通过 `AGENT_MIN_PAPERS` 调整）
  - 统计 `paper_register` 调用次数
  - 不满足则拦截并提示继续搜集

## 常见问题

### Q1: Agent 搜索不到论文怎么办？

**A**: 检查以下几点：
1. 确认网络连接正常
2. 检查 API Key 是否有效
3. 尝试更换数据库（arXiv → Semantic Scholar → OpenAlex）
4. 检查关键词是否过于具体，尝试更通用的关键词

### Q2: PDF 下载失败怎么办？

**A**:
1. 检查 arXiv ID 是否正确
2. 确认网络连接正常
3. 检查磁盘空间是否充足
4. 查看日志中的错误信息

### Q3: 笔记生成质量不高怎么办？

**A**:
1. 检查 PDF 是否能正常提取文本
2. 尝试调整 RAG 参数（Top-K 段落数）
3. 检查 Embedding 模型是否正常工作
4. 手动编辑笔记，然后让 Agent 基于修改后的笔记重写综述

### Q4: 如何添加新的学术数据库？

**A**:
1. 在 `tools/` 下创建新的工具文件
2. 继承 `core.tools.BaseTool`
3. 实现 `execute(**kwargs)` 方法
4. 在 `main.py` 中注册工具
5. 更新 README 文档

### Q5: 如何自定义 Agent 的行为？

**A**:
1. 修改 `prompts/` 目录下的 prompt 模板
2. 调整 `core/agent.py` 中的参数
3. 修改 `tools/` 中的工具逻辑
4. 调整环境变量（如 `AGENT_MIN_PAPERS`）

### Q6: 如何部署到生产环境？

**A**:
1. 使用 Docker 容器化
2. 配置 Nginx 反向代理
3. 使用 Gunicorn + Uvicorn 部署
4. 配置 HTTPS
5. 设置日志轮转
6. 配置监控和告警

### B. 环境变量说明

| 变量名 | 必需 | 默认值 | 说明 |
|--------|------|--------|------|
| `ZHIPU_API_KEY` | ✅ | - | 智谱 AI API Key |
| `ZHIPU_MODEL` | ✅ | glm-4-flash | 使用的 LLM 模型 |
| `SEMANTIC_SCHOLAR_API_KEY` | ❌ | - | Semantic Scholar API Key |
| `ARXIV_USER_AGENT` | ❌ | - | arXiv 请求 User-Agent |
| `AGENT_MIN_PAPERS` | ❌ | 3 | 质量门禁：最少收录论文数 |
| `AGENT_LOOP_DELAY_SEC` | ❌ | 3 | Agent 循环间冷却秒数 |

### C. 常用命令

```bash
# 启动 Web 服务
python web_app.py

# 运行 CLI 模式
python main.py

# 运行测试
python -m pytest tests/ -v

# 运行特定测试
python -m pytest tests/test_agent.py::test_chat_message_uses_ai_answer_for_normal_question -xvs

# 查看测试覆盖率
python -m pytest tests/ --cov=. --cov-report=html

# 代码格式化
black agent/

# 代码检查
flake8 agent/

# 类型检查
mypy agent/
```
