# 🎯 评估平台 (Eval Platform) 三周开发冲刺计划

本篇文档旨在为**评估平台开发团队（3人）**提供一份清晰、可量化、可执行的 3 周冲刺（Sprint）计划。目标是在三周内从零搭出一个能够接收 Agent 日志、运行 Ragas 评测并进行数据可视化的全栈 Web 系统。



---

## 🚀 Week 1：基建搭建与打通单点（4.16 - 4.22）
**核心目标：前后端脚手架搭建完毕，跑通本地 Ragas 评分打样。**

* **后端任务 (Backend)**
  * [ ] 初始化后端工程（推荐 FastAPI 或 Flask，轻量且好调 Python 算法库）。
  * [ ] 设计并实现数据库表结构（可使用 SQLite 快速起步）：
    * `agent_tasks` 表（记录每一次 Agent 运行的任务元数据）。
    * `agent_traces` 表（记录 Thought/Action 轨迹流）。
    * `evaluation_results` 表（记录算出来的打分结果）。
  * [ ] 开发一个 `POST /api/upload_json` 接口，能够接收 Agent 端传来的 JSON 格式日志并落库。

* **前端任务 (Frontend)**
  * [ ] 初始化 Vue3 / React 脚手架，引入组件库（ElementPlus / Ant Design）。
  * [ ] 搭建基础 Layout（左侧导航栏、顶部 Header）。
  * [ ] 实现第一个页面：**【测试任务列表页】**（以表格形式展示后端库里存了哪些 Agent 运行记录，暂无图表即可）。

* **算法/评测任务 (Eval Algorithm)**
  * [ ] 写一个独立的 Python 脚本，造一条虚假的 Agent JSON 数据。
  * [ ] 接入 `ragas` 库，成功基于这条假数据算出：`Faithfulness` (忠实度) 和 `Answer Relevance` (答案相关性)。
  * [ ] 将这个脚本打包成一个普通函数，准备下周给后端调用。

---

## 🔗 Week 2：核心评测接入与前后端联调（4.23 - 4.29）
**核心目标：让平台真正拥有“机器裁判”的能力，并且在前端能看到数据。**

* **后端任务 (Backend)**
  * [ ] 集成第一周写好的 Ragas 算分函数。由于 Ragas 算分较慢，建议做成**异步任务**（如通过简单的 background tasks），接收到 JSON 后在后台算分。
  * [ ] 开发获取评测结果的接口 `GET /api/evaluation/{task_id}`，包括各项分数和 Trace 数据。

* **前端任务 (Frontend)**
  * [ ] 引入图表库（ECharts / G2Plot）。
  * [ ] 开发核心页面：**【单次任务详情看板 Dashboard】**。
    * [ ] 上半部分：展示 Ragas 多维评分的雷达图、执行耗时卡片。
    * [ ] 下半部分：用卡片/时间轴形式复原 `trajectory`（Agent 搜索了啥、读了啥、发生了什么报错）。
  * [ ] 与后端联调，把数据从假数据替换为走网络请求的真数据。

* **算法/评测任务 (Eval Algorithm)**
  * [ ] 扩充评测维度：加入 **LLM-as-a-Judge（大模型裁判）**。
  * [ ] 编写一个判定脚本，让大模型（如 GPT-4 裁判）读一遍 JSON 里的 `trajectory`，打出【计划合理性】和【工具调用成功率】的分数。
  * [ ] 把这个新评审脚本也塞进后端的后台任务中。

---

## 🎨 Week 3：大规模测试、打磨与准备答辩（4.30 - 5.6）
**核心目标：UI 极致优化、消除潜在阻断 Bug，为最终验收生成“漂亮的数据”。**

* **全栈联合任务 (打磨与鲁棒性增强)**
  * [ ] **异常处理处理**：前端必须加上请求 Loading 动画；当 Ragas 评测超时报错时，不能导致页面白屏，应当能显示“评测失败，请重试”。
  * [ ] **批量对比页面（进阶加分项）**：加一个【版本对比】页面。选定同一个学术搜索词（比如：*LLM Memory*），分别展示“迭代一单Agent”与“迭代二双Agent”的成文分数对比，形成极具说服力的柱状图（体现项目价值）。
  
* **验收与汇报准备**
  * [ ] 让 Agent 团队疯狂跑 10 个以上不同的学术 Topics，把真实的 JSON 丢给系统，把系统的数据库丰富起来。
  * [ ] 完善当前文件夹下前端（`frontend/README.md`）和后端（`backend/README.md`）的启动运行说明，确保助教 clone 代码后能一行命令跑起来。
  * [ ] 录制平台的操作演示视频，导出最好看的图表截图，发给组长写进最终的答辩 PPT 中。
